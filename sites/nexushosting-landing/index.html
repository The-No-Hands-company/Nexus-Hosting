How it works

Simple idea.
Solid implementation.

01

Run a node

Any server with Docker becomes a Nexus node. It gets an Ed25519 keypair, joins the federation, and starts peering with other operators.

02

Deploy your site

Upload files through the dashboard or use nh deploy. Connect a Git repo for push-to-deploy. Static HTML or a live Node.js server — both work.

03

Replicate automatically

On every deploy, your node sends a signed site_sync to all federation peers. They pull the files. Visitors get routed to the closest node.

04

Nobody owns it

Each node is independently operated. No central authority. No company that can take your site offline. The network is as resilient as its operators.

For node operators

Run a node in 5 minutes

A $6/month VPS, a Raspberry Pi 4, a spare laptop — anything with Docker works. The full stack runs on 1 GB RAM with LOW_RESOURCE=true.

Quick start 
git clone https://github.com/The-No-Hands-company/Nexus-Hosting.git
cd Nexus-Hosting && cp .env.example .env

# Edit .env — five things to set:
# DATABASE_URL, ISSUER_URL, OIDC_CLIENT_ID,
# COOKIE_SECRET, PUBLIC_DOMAIN
# MinIO (S3 storage) is bundled — no external account needed

docker compose up -d

curl http://localhost:8080/api/health/live
# {"status":"ok"}
⬡

Join the federation

Set BOOTSTRAP_URLS to connect to existing peers on first start. Gossip protocol handles discovery automatically after that.

⚙

You set the rules

Declare your storage capacity. Set per-user caps if needed. Full abuse report system and IP banning built into the admin panel.
